## Outline

You are performing an AI-assisted customer report analysis. Your goal is to produce `profiles/customer-profile.md` — a concrete picture of who uses the product, at what scale, and how — so all implementation decisions are grounded in real customer data.

### Step 1 — Locate reports

- If user provided a path, use that directory.
- Check if `profiles/.customer-analyse-context.md` exists and read the source path.
- Scan for: `.csv`, `.json`, `.yaml`, `.yml` files.
- Common sources: Google Analytics exports, Mixpanel/Amplitude CSVs, CRM exports, support ticket exports, product analytics, cohort reports.

### Step 2 — Read and parse reports

For each file, extract:

**Scale metrics**:
- Total users / customers
- Daily Active Users (DAU), Monthly Active Users (MAU)
- Peak concurrent users
- Growth rate (if time-series available)
- Geographic distribution (if present)

**Usage patterns**:
- Most-used features / endpoints / actions
- Session length / frequency
- User journey / funnel data (if present)
- Retention / churn rate
- Feature adoption rates

**Customer model**:
- User segments (free/paid, role, plan tier, etc.)
- Power users vs casual users (usage distribution)
- Time-of-day / day-of-week patterns (if present)
- Common error scenarios / support patterns (if present)

### Step 3 — Synthesise the customer model

Describe:
1. **Who the primary users are** (persona sketch from data)
2. **How they use the product** (dominant flows)
3. **Scale at which the system must operate** (current + growth headroom)
4. **What matters most to them** (most-used features = highest value)
5. **Where they struggle** (errors, churn triggers, if detectable)
